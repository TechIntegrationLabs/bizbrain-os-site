Architecture

Clean Separation by Design

System files update via git pull. Your data and config are never touched.

Folder Structure 
bizbrain-os/
|
|-- .bizbrain/              # SYSTEM (git-tracked, auto-updated)
|   |-- modules/            #   27 module definitions (JSON configs)
|   |-- dashboard/          #   Local web UI (Node.js + vanilla JS)
|   |-- wizard/             #   Setup interview & generators
|   |-- agents/             #   AI agent definitions
|   |-- commands/           #   Slash command definitions
|   +-- templates/          #   Entity & project templates
|
|-- config.json             # YOUR CONFIG (gitignored)
|-- CLAUDE.md               # YOUR CONTEXT (AI-generated)
|-- Clients/                # YOUR DATA (gitignored)
|-- Partners/
|-- Vendors/
|-- Projects/
|-- Knowledge/
+-- Brand/                  # Logo, colors, assets
Update Without Fear 
$ git pull origin main
# System files update. Your data and config are untouched.

Frequently Asked Questions

Everything you need to know before getting started.

No. BizBrain OS is 100% local-first. All your business data stays on your machine. Nothing is uploaded to any cloud service. Your context files are gitignored by default, meaning they never leave your local workspace. The only data that touches external services is when you explicitly choose to use an AI provider (Claude, OpenAI, etc.) for processing.

Claude Code is the recommended engine for the best experience, but BizBrain OS is designed as a universal context layer. The structured files it generates (CLAUDE.md, entity records, knowledge base) can be fed into any LLM. You can use the context with ChatGPT, Cursor, Copilot, or any other AI tool that accepts text context.

A CRM stores contact data. BizBrain OS captures context — the nuances, preferences, communication styles, decision history, and institutional knowledge that make AI truly useful. It's the difference between knowing a client's email and knowing how they like to be communicated with, what projects have worked, and what they'll likely ask next. CRMs are for humans to look up data. BizBrain OS is for AI to understand your world.

Yes! The architecture supports both individual and team use. System files (modules, commands, templates) are git-tracked, so your team can share the same BizBrain OS setup. Each team member's personal data is gitignored. You can selectively share context by committing specific knowledge base files while keeping sensitive client data local. Many teams use a shared repo for the system files and individual workspaces for private data.

Updates are seamless. Run git pull origin main and system files update automatically. Your data, config, and context are never touched because they're in gitignored directories. New modules and features appear instantly. No migration scripts, no database changes, no breaking updates. Your context layer is yours forever.

BizBrain OS is open source under the AGPL v3 license. There's no catch, no freemium, no hidden fees. The project is built and maintained by Tech Integration Labs and the community. You'll need your own Claude Code subscription (or other AI provider) to use the AI features, but BizBrain OS itself is completely free. We believe the context layer should be an open standard, not a walled garden.
